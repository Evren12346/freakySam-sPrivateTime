MacBook Anonymizer
Main script literal walkthrough
File covered: bin/macbook_anonymizer.sh

Purpose of this document
- This file is meant to be read next to the source.
- It follows the script from top to bottom in source order.
- For executable logic, each meaningful line is called out directly.
- Blank lines are skipped because they do not affect runtime behavior.
- The one major exception is the big help-text here-document inside usage(); that block is mostly static text,
  so it is described as printed output instead of pretending each printed text line has separate program logic.

Legacy naming note
- The current repo name is MacBook Anonymizer.
- The script still contains older internal names such as "macbook anonymizer" and ~/.macbook-anonymizer.
- That does not change the flow of the code, but it does mean some printed labels and state paths still use the older name.

What the script does overall
- It installs and manages Tor on macOS.
- It snapshots the user's current proxy configuration before making changes.
- It reroutes macOS network services through Tor's SOCKS proxy.
- It can restore original proxy settings normally or through a panic path.
- It provides testing, diagnostics, hostname cloaking, and safer-app launching helpers.

Most important execution paths
- Normal start: `cmd_start()` saves state, starts Tor, waits for the SOCKS port, then routes services through Tor.
- Normal stop: `cmd_stop()` restores saved proxy state and stops Tor.
- Emergency stop: `cmd_panic_stop()` force-disables proxy paths first, then tries to restore and stop Tor.
- Guided verification: `cmd_self_test()` performs a start-like flow, checks routing, and restores state automatically on exit.

Main implementation caveats
- Internal strings and state-file names still use the old project name.
- JSON parsing is done with `sed`, which is simple but brittle.
- The script directly changes real macOS network settings, so testing should be done carefully.
